# 工作流模式 (Workflow Patterns)

## 顺序工作流

对于复杂任务，将操作分解为清晰的顺序步骤：

```markdown
创建新组件涉及以下步骤：

1. 确定组件类型 (atom/molecule/organism)
2. 创建组件文件 (`src/components/[type]/[name].tsx`)
3. 编写 TypeScript 类型定义
4. 实现组件逻辑
5. 添加 Storybook 示例（可选）
```

## 条件工作流

对于有分支逻辑的任务，引导通过决策点：

```markdown
1. 确定操作类型：
   **创建新内容?** → 跟随 "创建工作流"
   **编辑现有内容?** → 跟随 "编辑工作流"
   **删除内容?** → 跟随 "删除工作流"

2. 创建工作流:
   a. 验证输入 (Zod Schema)
   b. 调用 Repository
   c. 更新缓存
   d. 返回结果

3. 编辑工作流:
   a. 检查权限
   b. 验证输入
   c. 执行更新
   d. 触发 revalidation
```

## 决策树模式

```markdown
收到开发请求
├─ 涉及数据库? 
│   ├─ 是 → 使用 db-operations 技能
│   └─ 否 → 继续
├─ 涉及 UI?
│   ├─ 是 → 使用 ui-components 技能
│   └─ 否 → 继续
├─ 涉及认证?
│   ├─ 是 → 使用 auth 技能
│   └─ 否 → 使用通用开发流程
```

## 循环工作流

```markdown
## 迭代开发流程

重复以下步骤直到完成：

1. 实现一个小功能
2. 测试功能
3. 检查是否通过
   - 通过 → 继续下一个功能
   - 失败 → 修复问题，回到步骤 2
4. 提交代码
```
